# 智能网页信息采集 Agent

**大模型应用开发** | 2026.03 — 至今

## 项目简介

针对大模型后训练及 RAG 系统对高质量语料的需求，设计并实现了一套**基于 LLM 驱动的自愈式网页信息采集 Agent**。系统以 LangGraph 状态机为核心编排引擎，将网页抓取、DOM 分析、脚本生成、沙箱执行、错误诊断与代码自修复整合为闭环的 ReAct 自主循环。突破传统硬编码爬虫"换站即失效"的局限，实现跨网站、跨反爬策略、跨分页模式的零代码自适应数据提取。

**GitHub**: [handsome-bond/GraphSpider](https://github.com/handsome-bond/GraphSpider)

## 技术栈

`Python` `Asyncio` `Playwright` `LangGraph` `LangChain` `DeepSeek API` `RegEx` `JSON`

## 技术亮点

### 1. 自适应分页检测与多形态网页适配

系统通过静态 HTML 分析自动识别网站分页类型——URL 参数跳转型、AJAX 按钮点击型、瀑布流追加型——并针对替换式分页（政府/企业网站常见，每页条目数不变）设计指纹比对算法，以首条数据文本变化作为翻页成功判据，替代传统 count 增量检测，彻底解决"爬了第一页就假死"的行业痛点。同时针对动态 AJAX 渲染页面与传统静态网页自动切换不同的等待策略和 DOM 稳定化机制，将 403 封禁率降至接近零。

### 2. 11 级后处理代码修复管道

LLM 生成代码后，系统执行一套完整的后处理修复链：URL 幻觉回正、count 检测→指纹检测转换、JS 字符串注入修复（`json.dumps` 转义）、环境变量名校正、Bootstrap disabled 位置修正、骨架屏空节点过滤、代理自动注入、持久化浏览器上下文替换等。这套管道使生成脚本的首次可用率从 ~30% 提升至 ~80%，是 Agent 自愈循环的"快速通道"。

### 3. 两级自愈策略与错误分类

Agent 内置正则驱动的错误分类器，覆盖 TIMEOUT、HTTP 403、JS SyntaxError、NameError、EMPTY_RESULT、LOGIN_REQUIRED 等 8 种错误模式。修复采用两级策略：语法/变量错误走 sanitizer 管道（零 LLM 调用，毫秒级修复）；选择器/反爬错误触发全管道重新生成（带错误上下文注入 prompt）。最多 3 轮自动重试，登录页检测到后直接终止不浪费 API 配额。

### 4. 交互式登录与持久化会话

针对知乎等强制登录站点，设计了持久化浏览器上下文方案——基于 Playwright `launch_persistent_context` 维护真实 Chromium 用户目录，首次运行时弹出浏览器供用户扫码登录，此后所有会话自动复用，无需手动提取 Cookie。登录态天然持久化，不受 `storage_state` 序列化缺陷影响。

### 5. CSS 选择器防幻觉注入

从抓取的 HTML 中实时提取所有 CSS class 名称和 ID，以结构化列表形式注入 LLM 的生成 prompt，强制模型只能使用页面上真实存在的选择器。配合头-中-尾三段式 DOM 分块采样策略，确保数据区 HTML 不会因关键词过滤被误丢弃，彻底消除大模型"凭空编造选择器"的幻觉问题。

### 6. 高信噪比数据清洗与标准化落盘

设计正则表达式与 DOM 节点双重过滤机制——自动剔除隐藏骨架屏空节点、去除非结构化文本中的冗余换行符与前端噪声——提取核心业务字段输出为标准 JSON/CSV/JSONL 格式，为下游 RAG 知识库或后训练合成数据提供高品质、零污染的数据源。
